# Contributing to Teknofest UGV

Thank you for your interest in contributing to our Teknofest UGV project!

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/UGV-main.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit: `git commit -m "Add your feature"`
7. Push: `git push origin feature/your-feature-name`
8. Create a Pull Request

## Code Style

### Python (ROS 2 Nodes)
- Follow PEP 8
- Use type hints where appropriate
- Keep functions focused and small
- Add docstrings for public functions

### C++ (Teensy Firmware)
- Follow Google C++ Style Guide
- Use meaningful variable names
- Comment complex logic
- Keep FreeRTOS tasks simple

## Testing

Before submitting a PR:
- Test on actual hardware if possible
- Verify ROS 2 nodes build without errors
- Check firmware compiles for Teensy 4.1
- Run system integration tests

## Pull Request Guidelines

- Describe what your PR does
- Reference any related issues
- Include test results
- Update documentation if needed

## Questions?

Open an issue or contact the team members listed in README.md
